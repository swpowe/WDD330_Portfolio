Exception handling, validating

week05-notes and reading

Probably one of the most important things to implement in language writing is validation,
error or exception handling, this concept is centered around making sure that whenever you
input incorrect data it will not shut down or cause the program to error out. This for
obvious reason makes it so that when you input a 15 instead of fifteen it will handle
this exception and give you a message instead of shutting down. There are many ways for
you to do this, but my favorites are validation, Exceptions handling, and testing.

  1. Validation is simple, it is when we take and write a function that will not allow
    certain information through, or will tell it is not valid, like the example explained above.
  2. Exception handling is when we take a function and we tell it that we would like to try
    something, and if it fails then we do something else. Essentially you are making sure it
    doesn’t not stop the program mid-way through. And will allow you to see the full program
    running, even if it doesn’t work fully.
  3. Testing is simple in nature to, to test means to write maybe a basic function that will
    make sure that certain variable or data works. For example. If I make an object array, I
    can run a print log to make sure the info was stored correctly.

To conclude, I think I understand the nature of this section well. In truth its more
just learning more techniques to test your code and keep the everything running smoothly.
So for me to excel in this section, it would be good to learn more about what kind of
validation I could use.
