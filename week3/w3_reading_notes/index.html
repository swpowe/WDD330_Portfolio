Objects, DOM's, and Events

week03-notes and reading

I thought that this week reading was very interesting, the three major things
I learned were about what objects are and how they are defined by elements,
DOM’s document object model which are almost a type of identifier which allows
you to pull information from the object, and finally events, which are what
triggers the usage of the DOM and then using them in some way.

  1. Objects are… well objects really, they are entities with labels behind them.
    an example would be with a object I have a name or title, like “dog” and then
    I have defining attributes like breed: shepherd, color: black, white, food
    preference: meat these defining “elements” allow us to call the object up and
    use these defining elements
  2. DOM which stands for document object model, is a type of formation that allows
    us to call up the object itself, this is done in many different ways, but
    ultimately you are trying to grab the elements inside the object and organize
    them in some way so they are ready to be used.
  3. Events are the final trigger so to speak. Events are activation keys that will
    allow the DOM’s to be activated for example this might include some sort of
    console.log or maybe some other element that says hey I have been activated pull
    the information. Buttons are other forms of events more on the GUI spectrum, but
    it is something that will make the info inside the object do what ever was defined
    by the DOM.
